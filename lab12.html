1.1. ¿Qué es un Área de Endemismo? (La regla de múltiples especies)

La Biogeografía Histórica estudia cómo ha evolucionado la distribución de los seres vivos a través del tiempo. Su unidad básica es el Área de Endemismo (AoE).

Un Área de Endemismo se define como una región donde los mapas de distribución de dos o más especies diferentes coinciden y esas especies no se encuentran en ninguna otra parte del mundo.

⚠️ ¿Por qué necesitamos muchas especies diferentes?
Si una sola especie de rana vive únicamente en una montaña, esa montaña es el área de distribución de la rana, pero NO es un Área de Endemismo histórica. Buscamos patrones. Si en esa misma montaña coinciden una rana, un escarabajo y una orquídea exclusivas, tenemos evidencia real de que un evento del pasado (como el nacimiento de esa montaña) aisló a múltiples seres vivos a la vez. Por esto, su análisis debe incluir múltiples especies del grupo que investiga.

1.2. ¿Cómo evalúa el programa los mapas? (Szumik & Goloboff, 2004)

Para encontrar estas áreas, usamos el programa NDM/VNDM. El programa dibuja una cuadrícula (Grid) sobre el mapa y calcula un "Índice" de puntaje para diferentes grupos de celdas:

  • Si una especie está en todas las celdas de un grupo, y no existe por fuera, suma 1 punto perfecto al área.
  • Si la especie falta en algunas celdas, el programa suma menos de 1 punto (asume que tal vez los biólogos no buscaron bien ahí).
  • Si la especie "se escapa" y vive muy lejos en el mapa, el programa la castiga restándole puntos, porque ya no es exclusiva del área.

Si un grupo de celdas suma un puntaje de 2.0 o más, el algoritmo lo aprueba como un Área de Endemismo válida (porque matemáticamente equivale al peso de al menos dos especies endémicas perfectas).

3. Tutorial Paso a Paso: De R a VNDM

I
Fase I: Minería y Limpieza Automática en R

El manual oficial de NDM/VNDM estipula que el programa falla si el archivo contiene nombres de especies con espacios, si tiene encabezados (títulos de columnas) o si incluye comas. El siguiente script asegura un formato perfecto.

# ==============================================================================
# LAB 12: MINERÍA Y CURACIÓN DE DATOS ESPACIALES PARA VNDM
# ==============================================================================

# 0. Configurar el Directorio de Trabajo (Workspace)
# Use su ruta exacta. En Windows se recomiendan barras hacia adelante (/).
setwd("C:/Ruta/A/Su/Carpeta_de_Sistematica") 

# 1. Instalar paquetes necesarios (¡Ejecute esta línea solo una vez en su computador!)
install.packages(c("rgbif", "dplyr", "stringr", "CoordinateCleaner"))

# 2. Cargar las librerías para usarlas en esta sesión
library(rgbif)
library(dplyr)
library(stringr)
library(CoordinateCleaner)

# 3. Obtener el Código Taxonómico Oficial (Taxon Key)
# Esto evita que GBIF se confunda con el texto del nombre.
print("Buscando el código taxonómico en GBIF...")
info_taxon <- name_backbone("Espeletia") # Cambie "Espeletia" por su clado
codigo_gbif <- info_taxon$usageKey

print(paste("Código de GBIF encontrado:", codigo_gbif))

# 4. Descarga de Datos Crudos usando occ_search
print("Descargando registros... Esto puede tomar un minuto.")
registros <- occ_search(
  taxonKey = codigo_gbif,               
  country = "CO",                       
  basisOfRecord = "PRESERVED_SPECIMEN", # Exige vouchers físicos de museo
  hasCoordinate = TRUE,                 
  limit = 5000                          
)

datos_crudos <- registros$data

# Validación de seguridad
if(is.null(datos_crudos) || nrow(datos_crudos) == 0) {
  print("Error: No se encontraron registros. Revise el nombre o la conexión.")
} else {
  print(paste("Registros iniciales:", nrow(datos_crudos)))
  
  # 5. Filtro Básico
  datos_filtrados <- datos_crudos %>%
    filter(!is.na(decimalLongitude), !is.na(decimalLatitude), !is.na(species))
  
  # 6. Curación Espacial (CoordinateCleaner)
  print("Iniciando curación espacial...")
  datos_curados <- clean_coordinates(
    x = datos_filtrados,
    lon = "decimalLongitude",
    lat = "decimalLatitude",
    tests = c("institutions", "zeros", "seas"), # Elimina océanos y museos
    value = "clean"             
  )
  
  # 7. Formato Estricto para VNDM
  # VNDM se corrompe si el taxón tiene espacios. Se reemplazan por guiones bajos.
  # NDM exige que el archivo no contenga separadores por comas.
  datos_vndm <- datos_curados %>%
    mutate(species = str_replace_all(species, " ", "_")) %>%
    select(decimalLongitude, decimalLatitude, species) %>%
    distinct()
  
  print(paste("Registros listos para exportar:", nrow(datos_vndm)))
  
  # 8. Exportación Final
  # El archivo NO debe contener títulos de columnas (col.names = FALSE).
  # Las columnas deben estar separadas por tabulaciones (sep = "\t").
  write.table(datos_vndm, "ocurrencias_vndm.xyd", 
              sep = "\t", row.names = FALSE, col.names = FALSE, quote = FALSE)
  
  print("¡Éxito! El archivo ocurrencias_vndm.xyd ha sido creado.")
}
II
Fase II: Cargar datos en VNDM
  1. Abra el programa vndm.exe. Ambos programas (NDM y VNDM) deben estar en la misma carpeta.
  2. Vaya al menú superior: File → Read XYZ / Data point format.
  3. Seleccione su nuevo archivo ocurrencias_vndm.xyd.
  4. Aparecerá un cuadro de diálogo. Seleccione la opción "X, Y, Sp" (Longitud, Latitud, Especie). Haga clic en OK.
  5. La pantalla negra dibujará el mapa y mostrará sus especies como puntos de colores. El panel izquierdo mostrará las especies leídas.
III
Fase III: Crear la Cuadrícula (Grid)

Usted puede usar las teclas de dirección del teclado para ajustar rápidamente el tamaño y origen de la grilla.

  1. Vaya al menú Grid → Set origin and sizes.
  2. Busque la sección Cell Size (X/Y). Aquí define el tamaño en grados de sus cuadros.
  3. Escriba 1.0 en X y 1.0 en Y (cuadros de ~111 km, estándar para analizar a nivel de país). Si su clado vive en una zona muy pequeña, intente con 0.5.
  4. Haga clic en OK. La malla aparecerá sobre el mapa. Las celdas con registros tendrán un punto rojo.
  5. Ajustes Rápidos (Atajos de teclado):
    • Use ↑ ↓ ← → para cambiar rápidamente el tamaño de la celda.
    • Use Ctrl + ↑ ↓ ← → para desplazar el origen de la grilla sobre el mapa.
IV
Fase IV: Búsqueda del Algoritmo (NDM)

El programa NDM actuará como motor de búsqueda matemático llamado desde VNDM.

  1. Vaya al menú Analyze → Find areas (Heuristic search).
  2. Ajuste estos tres valores en la ventana de características de búsqueda:
    • Save sets with score > Escriba 2.0.
    • Swap one cell at a time: Marque esta casilla para afinar los bordes del área (hace más lenta la búsqueda).
    • Replications: Escriba 100 para asegurar una búsqueda completa.
  3. Haga clic en Search.
V
Fase V: Agrupar resultados en un Consenso

A menudo el programa encuentra áreas que difieren por apenas una celda o especie. Las agruparemos para resumir la información.

  1. Vaya a Analyze → Consensus areas....
  2. En la ventana "Consense areas with cut-off (% similarity in species)", escriba 50.
  3. Asegúrese de seleccionar la opción "against each of the other areas in the consensus" (Consenso estricto). Haga clic en OK.
VI
Fase VI: Resultados y Exportación
  1. Vaya al menú View y seleccione Show areas (o use las teclas para avanzar si los resultados se muestran solos).
  2. Use las teclas Enter y Backspace para avanzar o retroceder en la lista de áreas o especies (según lo que esté visualizando).
  3. Lectura de Datos (Para su informe):
    • Score: Es el puntaje matemático de su área. Entre más alto, mejor.
    • Lista inferior: Muestra los nombres de las especies exclusivas de esa zona. Presione V + Enter para ver la lista desglosada de los taxones endémicos que aportan al índice del área. Anote los nombres.
  4. Para exportar la imagen a su documento, presione Shift + M o F12 para guardar el área en pantalla como un Metafile (*.emf).
