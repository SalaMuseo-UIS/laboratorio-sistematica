1.1. ¿Qué es un Área de Endemismo? (La regla de múltiples especies)

La Biogeografía Histórica estudia cómo ha evolucionado la distribución de los seres vivos a través del tiempo. Su unidad básica es el Área de Endemismo (AoE).

Un Área de Endemismo se define como una región donde los mapas de distribución de dos o más especies diferentes coinciden y esas especies no se encuentran en ninguna otra parte del mundo.

⚠️ ¿Por qué necesitamos muchas especies diferentes?
Si una sola especie de rana vive únicamente en una montaña, esa montaña es el área de distribución de la rana, pero NO es un Área de Endemismo histórica. Buscamos patrones. Si en esa misma montaña coinciden una rana, un escarabajo y una orquídea exclusivas, tenemos evidencia real de que un evento del pasado (como el nacimiento de esa montaña) aisló a múltiples seres vivos a la vez. Por esto, su análisis debe incluir múltiples especies del grupo que investiga.

1.2. ¿Cómo evalúa el programa los mapas? (Szumik & Goloboff, 2004)

Para encontrar estas áreas, usamos el programa NDM/VNDM. El programa dibuja una cuadrícula (Grid) sobre el mapa y calcula un "Índice" de puntaje para diferentes grupos de celdas:

  • Si una especie está en todas las celdas de un grupo, y no existe por fuera, suma 1 punto perfecto al área.
  • Si la especie falta en algunas celdas, el programa suma menos de 1 punto (asume que tal vez los biólogos no buscaron bien ahí).
  • Si la especie "se escapa" y vive muy lejos en el mapa, el programa la castiga restándole puntos, porque ya no es exclusiva del área.

Si un grupo de celdas suma un puntaje de 2.0 o más, el algoritmo lo aprueba como un Área de Endemismo válida (porque matemáticamente equivale al peso de al menos dos especies endémicas perfectas).

El manual oficial del algoritmo especifica que el formato *.xyd tiene una arquitectura milimétrica. Para evitar corrupciones y colapsos del programa, utilizaremos una función robusta en R para construir el documento de forma segura.

# 2. Función Robusta para Crear Archivos .xyd (Basado en jupanago)
write_xyd <- function(data, filename) {
  # Extraer especies únicas y cantidad total
  species_list <- unique(data$species)
  n_spp <- length(species_list)
  
  # Iniciar el archivo y escribir los encabezados estrictos de la V3.1
  sink(filename)
  cat("nocommas\n")
  cat("longlat\n")
  cat(paste0("spp ", n_spp, "\n"))
  cat("xydata\n")
  
  # Bucle para imprimir cada especie y sus coordenadas asociadas
  for (i in seq_along(species_list)) {
    sp <- species_list[i]
    # VNDM numera desde 0 y requiere corchetes exactos
    cat(paste0("sp ", i - 1, " [", sp, "]\n"))
    
    # Extraer y formatear coordenadas
    coords <- data %>% 
      filter(species == sp) %>% 
      select(decimalLongitude, decimalLatitude)
    
    # Imprimir coordenadas línea por línea
    for (j in 1:nrow(coords)) {
      cat(paste(coords$decimalLongitude[j], coords$decimalLatitude[j], "\n"))
    }
  }
  sink()
  print(paste("Archivo", filename, "creado con éxito. Contiene", n_spp, "especies."))
}

# 3. Descarga de Datos desde GBIF usando Taxon Key
print("Buscando código taxonómico...")
info_taxon <- name_backbone("Espeletia") # Cambie "Espeletia" por su clado
codigo_gbif <- info_taxon$usageKey

print("Descargando registros de museo (vouchers)...")
registros <- occ_search(
  taxonKey = codigo_gbif,               
  country = "CO",                       
  basisOfRecord = "PRESERVED_SPECIMEN",
  hasCoordinate = TRUE,                 
  limit = 5000                          
)

datos_crudos <- registros$data

# 4. Limpieza y Curación Espacial
if(is.null(datos_crudos) || nrow(datos_crudos) == 0) {
  print("Error: No se encontraron registros.")
} else {
  datos_filtrados <- datos_crudos %>%
    filter(!is.na(decimalLongitude), !is.na(decimalLatitude), !is.na(species))
  
  print("Eliminando coordenadas erróneas...")
  datos_curados <- clean_coordinates(
    x = datos_filtrados,
    lon = "decimalLongitude",
    lat = "decimalLatitude",
    tests = c("institutions", "zeros", "seas"), 
    value = "clean"             
  )
  
  # Limpiar espacios en nombres y seleccionar columnas finales
  datos_limpios <- datos_curados %>%
    mutate(species = str_replace_all(species, " ", "_")) %>%
    select(decimalLongitude, decimalLatitude, species) %>%
    distinct()

  # 5. Ejecutar la función para crear el archivo final
  write_xyd(datos_limpios, "ocurrencias.xyd")
}

Fase III: Crear y Ajustar la Cuadrícula (Grid)

El algoritmo necesita dividir el continente en cuadros para calcular las densidades y presencias.

  1. Vaya al menú Settings → Set grid size and position.
  2. En la sección Cell Size (X/Y) defina el tamaño en grados. Escriba 1.0 en X y 1.0 en Y (cuadros de ~111 km, estándar para analizar a nivel de país). Haga clic en OK.
  3. La malla aparecerá sobre el mapa. Atajos de teclado rápidos:
    • Use ↑ ↓ ← → para cambiar el tamaño de las celdas visualmente.
    • Use Ctrl + ↑ ↓ ← → para desplazar la grilla y acomodarla sobre el centro de sus puntos.
IV
Fase IV: Generar la Matriz Binaria (¡Paso Obligatorio!)

Antes de buscar áreas, debemos ordenarle al programa que cruce la cuadrícula con las coordenadas de las especies.

  1. Vaya al menú File.
  2. Haga clic en la opción Create matrix from data points.
  3. La pantalla cambiará, mostrando una cuadrícula verde con el título Set 0 of 1 (size=0). Score=0.000000. ¡No se asuste! Este es simplemente su lienzo en blanco. El programa agrupó dónde hay datos (las celdas verdes), preparándose para el análisis matemático.
V
Fase V: Ejecutar la Búsqueda Heurística (Analyze)

El menú Analyze es el "cerebro matemático" de VNDM. Como es un algoritmo de aproximación (heurístico), debemos configurarlo para que busque inteligentemente a través del mapa.

  1. Vaya al menú Analyze → Analyze with NDM.
  2. Verá una ventana con múltiples parámetros matemáticos. Asegúrese de ajustar o confirmar los siguientes valores críticos:
    • Save sets with [2] or more endemic spp.: Déjelo en 2. Esto garantiza que se cumpla la regla de oro de la biogeografía (un Área de Endemismo requiere la congruencia de al menos dos especies exclusivas).
    • Save sets with score above: Escriba 2.000. Esto le dice al programa que solo guarde ecosistemas que matemáticamente pesen lo mismo o más que dos especies endémicas perfectas.
    • Type of swapping: Seleccione swap one cell at a time. Esto define cómo el programa "dibuja" el área, añadiendo o quitando de a un cuadro a la vez para afinar cuidadosamente los bordes del ecosistema sin saltarse detalles geográficos.
    • Keep overlapping subsets if [0] % of species unique: Déjelo en 0. Significa que, temporalmente, queremos que el programa guarde todas las áreas superpuestas (incluso si son redundantes), porque en el siguiente paso nosotros mismos las limpiaremos haciendo un mapa de Consenso.
    • Repeat search... times: Escriba 100. El algoritmo empezará desde puntos aleatorios 100 veces diferentes para asegurarse de que exploró todas las combinaciones posibles de celdas en el mapa.
  3. Haga clic en el botón OK. Verá una barra de progreso indicando la evaluación matemática de las celdas.
VI
Fase VI: Consenso de Áreas

A menudo el programa encuentra versiones casi idénticas del mismo ecosistema, desplazadas por apenas una o dos celdas. En esta fase limpiaremos la redundancia sintetizando la información compartida.

  1. Vaya al menú Analyze → Consense areas....
  2. Verá una ventana pequeña con el título "Consense areas with cut-off". Aquí configuraremos las reglas de agrupación:
  3. % similarity in species: Cambie el valor de 100 (el predeterminado) a 50. Esto le dice al programa que agrupe las áreas identificadas si comparten al menos la mitad (50%) de sus especies endémicas exclusivas.
  4. Include an area if cut-off is ok...: Observe las dos opciones disponibles:
    • against each of the other areas in the consensus: Este es el Consenso Estricto. Solo agrupará un área si esta cumple la regla del 50% con todas y cada una de las demás áreas del grupo. Es el método más riguroso y exacto. (Seleccione esta opción).
    • against any of the other areas in the consensus: Este es el Consenso Flexible. Agrupa el área si cumple la regla con al menos una de las áreas del grupo. Suele generar áreas de consenso gigantescas y poco precisas, por lo que lo evitaremos en este laboratorio.
  5. Una vez confirmados el `50` y la opción `against each`, haga clic en OK.
VII
Fase VII: Exploración de Resultados (Actions)

El menú Actions es donde usted extrae la información biológica clave para su informe de laboratorio.

1. Lectura del Mapa de Consenso:

Al generar el consenso, notará que el mapa de celdas verdes cambia a una paleta de colores rojos y aparece nueva información vital en pantalla:

  • Título superior (ej. from 5 areas; max. values): Le indica cuántas áreas individuales similares fueron fusionadas matemáticamente para crear este mapa resumen.
  • Gradiente de Colores (Leyenda derecha): Muestra los rangos del Índice de Endemicidad (Score). Las celdas rojas oscuras representan el "núcleo" geográfico del área de endemismo (donde el puntaje es más alto y la coincidencia de especies es perfecta), mientras que los tonos rosados más claros son las zonas periféricas de ese ecosistema con menor soporte. Las celdas doradas ("Just ground!") son cuadrículas que tienen datos pero no pertenecen a esta área.
  • Lista inferior (Areas included): Le muestra los números de identificación exactos de las áreas individuales originales (ej. 3, 5, 6, 8, 11) que el algoritmo agrupó en este mapa.

2. Navegación:

Use las teclas Enter y Backspace para avanzar y retroceder a través de todas las diferentes áreas Consenso que descubrió el programa para Colombia.

3. Lectura del Desglose de Especies (Scoring species):

Mientras observa un área importante en su pantalla, vaya al menú Actions → Show scoring species (o presione V y luego Enter). Esto le revelará exactamente qué especies construyen esa zona biológica:

  • Encabezado individual: Arriba leerá cuánto puntaje aporta la especie que está viendo al total del ecosistema (ej. contributes 0.812500 to score=12.53).
  • Mapa Específico: Los puntos amarillos brillantes son las colectas reales de herbario/museo de esa especie. Las celdas rojas muestran dónde la especie coincide con el Área Endémica.
  • Estadísticas (Inside / Outside): Detalla cuántas celdas tiene la especie "adentro" (Inside: pres) frente a cuántas tiene "afuera" (Outside: obs). ¡Recuerde! Si la especie tiene presencias afuera (Outside > 0), el programa la penaliza y le reduce su puntaje porque pierde exclusividad.
  • Lista Inferior (Scoring spp.): Le entrega el listado completo y definitivo de las especies que sustentan su área de endemismo (ej. 19 scoring spp). Tome nota cuidadosa de estos nombres para su informe. Puede usar Enter para navegar el mapa individual de cada una de ellas.

4. Exportación Rápida de Imagen:

Para guardar el mapa como una imagen y anexarla a su informe, vaya a File → Save diagram as metafile (o presione F12).

VIII
Fase VIII: Exportación a Sistemas de Información Geográfica (QGIS)

Los cuadros de VNDM sobre un fondo blanco son útiles estadísticamente, pero dificultan saber qué departamento o cordillera exacta estamos analizando. Afortunadamente, la versión 3.1 nos permite exportar los píxeles a un mapa real[cite: 2].

  1. Mientras observa su mejor Área de Endemismo Consenso en la pantalla de VNDM, vaya al menú File.
  2. Seleccione Save coordinates for GIS → Save area/s as CSV. Guarde el archivo en su carpeta lab12[cite: 2].
  3. Abra QGIS en su computador. Añada un mapa base de Colombia (ej. OpenStreetMap o Google Terrain).
  4. Vaya al menú superior de QGIS: Capa → Añadir capa → Añadir capa de texto delimitado....
  5. Seleccione su nuevo archivo CSV. Asegúrese de indicar que el campo X es la Longitud (Long) y el campo Y es la Latitud (Lat).
  6. Al cargarla, la zona de endemismo matemática se proyectará perfectamente sobre la geografía real del país, permitiéndole identificar ríos, montañas o valles asociados a la distribución de sus especies. ¡Use esta imagen para fortalecer la discusión de sus resultados!
