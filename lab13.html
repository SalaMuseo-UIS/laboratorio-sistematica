Objetivos de Aprendizaje

  3. Interpretar el significado biológico de los parámetros de evolución espacial (dispersión continua, extinción y dispersión fundadora o salto).
  4. Seleccionar el mejor modelo evolutivo utilizando el Criterio de Información de Akaike (AICc).
  5. Analizar la reconstrucción gráfica (gráfico filogenético con proporciones) para identificar el área geográfica de origen de un clado.

1.1. Patrones vs. Mecanismos

En biogeografía, el análisis de patrones responde a la pregunta ¿Qué está dónde?. Su objetivo es agrupar áreas basándose en las especies que comparten. Para esto usaremos la plataforma Infomap Bioregions.

Por otro lado, la inferencia de mecanismos responde a ¿Cómo llegaron allí?. Busca explicar los procesos históricos (como dispersión o vicarianza) que generaron ese patrón. Para esto usaremos el programa estadístico BioGeoBEARS en RStudio.

1.2. Modelos Evolutivos en BioGeoBEARS

BioGeoBEARS calcula la probabilidad de que ocurran diferentes eventos a lo largo de un árbol evolutivo. En este laboratorio evaluaremos dos modelos:

  • Modelo DEC (Dispersión y Extinción): Asume que los organismos expanden su territorio lentamente hacia áreas vecinas (parámetro d) o se extinguen de manera local (parámetro e).
  • Modelo DEC+J (Dispersión Fundadora): Incluye el parámetro "J" (Jump dispersal o salto). Este parámetro asume que un individuo puede cruzar una barrera difícil (como un océano) y fundar de inmediato una nueva especie en una zona lejana. Es el modelo más común para explicar la vida en islas oceánicas.

1.3. Selección de Modelos (El valor AICc)

Para saber cuál modelo explica mejor la realidad biológica, usamos el Criterio de Información de Akaike (AICc). Este indicador hace un balance entre qué tan bien se ajusta el modelo a los datos (Verosimilitud o LnL) y la cantidad de parámetros que usa (Complejidad). La regla es simple: el modelo con el número de AICc más bajo es el ganador.

¿Qué datos consume BioGeoBEARS?
A diferencia de los mapas de distribución que usan coordenadas (latitud y longitud), los modelos evolutivos estocásticos requieren dos archivos sincronizados:
  1. Un Cronograma (.newick): Un árbol filogenético donde la longitud de las ramas representa tiempo real (millones de años). Esto es necesario para que el programa calcule la velocidad a la que ocurren los cambios.
  2. Una Matriz Geográfica (.data): Un archivo de texto (formato PHYLIP) que lista las especies y usa códigos binarios para indicar en cuáles áreas geográficas viven (1 = Presente, 0 = Ausente). Por ejemplo, 1000 significa que la especie solo vive en la primera área de la lista.

1.4. Visualización Gráfica en BioGeoBEARS

Además de generar tablas numéricas con las probabilidades, el programa dibuja un archivo PDF con la filogenia. En las puntas del árbol (hojas) se muestran las áreas actuales donde vive cada especie. En las uniones de las ramas (nodos internos) se dibujan gráficos de torta (pie charts).

Estos gráficos representan la probabilidad calculada por el modelo de que el ancestro haya habitado en una región específica. Si miras el nodo más profundo del árbol (la raíz), el color predominante te indicará de dónde proviene originalmente todo el grupo de especies. Además, el título principal del gráfico PDF suele mostrar un resumen de las tasas del modelo (por ejemplo, d=0, e=0, j=0.11) confirmando bajo qué condiciones se generó el dibujo.

Fase I: Minería de Datos

Este script descarga exclusivamente datos del género Psychotria delimitados por las coordenadas del archipiélago de Hawái.

# ==============================================================================
# LAB 13: FASE I - MINERÍA ESPACIAL DELIMITADA Y CURACIÓN
# ==============================================================================
setwd("C:/Ruta/A/Su/Carpeta") # Ajuste a su ruta exacta

library(rgbif)
library(dplyr)
library(stringr)
library(CoordinateCleaner)

# Obtener código del género
info_taxon <- name_backbone("Psychotria") 
codigo_gbif <- info_taxon$usageKey

# Descarga delimitada por coordenadas (Hawái)
print("Descargando registros...")
registros <- occ_search(
  taxonKey = codigo_gbif,               
  hasCoordinate = TRUE,
  decimalLatitude = "18,22.5",       
  decimalLongitude = "-160.5,-154",  
  basisOfRecord = "PRESERVED_SPECIMEN", 
  limit = 5000                          
)

datos_crudos <- registros$data

# Curación y Limpieza
datos_filtrados <- datos_crudos %>%
  filter(!is.na(decimalLongitude), !is.na(decimalLatitude), !is.na(species))
  
datos_curados <- clean_coordinates(
  x = datos_filtrados,
  lon = "decimalLongitude",
  lat = "decimalLatitude",
  tests = c("institutions", "zeros", "seas"), 
  value = "clean"             
)
  
datos_infomap <- datos_curados %>%
  mutate(species = str_replace_all(species, " ", "_")) %>%
  select(decimalLatitude, decimalLongitude, species) %>%
  distinct()
  
# Exportar
write.csv(datos_infomap, "ocurrencias_psychotria_hawaii.csv", row.names = FALSE)
print("¡Archivo creado exitosamente!")
II
Fase II: Parametrización en Infomap

Ingrese a Infomap Bioregions y siga estos pasos para agrupar las áreas basándose en las especies que contienen, filtrando posibles errores.

  1. Haga clic en Load data... y cargue su archivo CSV.
  2. En el cuadro "Select how to parse columns" asigne los menús así: Name = species, Latitude = decimalLatitude, y Longitude = decimalLongitude. Presione Submit.
  3. En la sección Resolution asegúrese de que Unit sea degree.
  4. Configure Max cell size a 1°.
  5. Escriba 1/8° en Min cell size. Permitir celdas pequeñas asegura que el programa separe islas vecinas que están divididas por mar.
  6. Configure Min cell capacity en 10. Esto elimina registros aislados y estabiliza el mapa.
  7. Desplácese a Clustering y presione Run... para ver el mapa de biorregiones.
  8. Use el botón Export... en la esquina inferior izquierda. Descargue una imagen de su mapa (Save map) y descargue el archivo _summary.csv (de la opción Save tables).

PARTE 2: Inferencia de Mecanismos (BioGeoBEARS)

III
Fase III: Ejecución de Modelos en R

Abra una sesión limpia en RStudio. Ejecutaremos el modelo continuo (DEC) y el modelo de dispersión fundadora (DEC+J).

# ==============================================================================
# LAB 13: FASE III - RECONSTRUCCIÓN DE ÁREAS ANCESTRALES
# ==============================================================================

setwd("C:/Cambie/Por/Ruta/A/Su/Carpeta")

library(BioGeoBEARS)
library(ape)

# 1. Cargar Archivos (Incluidos como ejemplo en la librería)
extdata_dir = np(system.file("extdata", package="BioGeoBEARS"))
trfn = paste(extdata_dir, "/Psychotria_5.2.newick", sep="")
geogfn = paste(extdata_dir, "/Psychotria_geog.data", sep="")

tr = read.tree(trfn)
tipranges = getranges_from_LagrangePHYLIP(lgdata_fn=geogfn)

# ==============================================================================
# 2. MODELO DEC (Dispersión y Extinción clásica)
# ==============================================================================
BioGeoBEARS_run_object = define_BioGeoBEARS_run()
BioGeoBEARS_run_object$trfn = trfn
BioGeoBEARS_run_object$geogfn = geogfn

# Límite biológico: una especie no puede ocupar más de 4 islas al mismo tiempo.
BioGeoBEARS_run_object$max_range_size = 4    
BioGeoBEARS_run_object$min_branchlength = 0.000001
BioGeoBEARS_run_object$include_null_range = TRUE
BioGeoBEARS_run_object$num_cores_to_use = 1 # Previene fallos en Windows

print("Optimizando modelo DEC...")
resDEC = bears_optim_run(BioGeoBEARS_run_object)

# Extraer parámetros estadísticos
stats_DEC <- extract_params_from_BioGeoBEARS_results_object(results_object = resDEC, returnwhat = "table", addl_params = c("j"))

# ==============================================================================
# 3. MODELO DEC+J (Incluye dispersión por saltos)
# ==============================================================================
BioGeoBEARS_run_object_J = define_BioGeoBEARS_run()
BioGeoBEARS_run_object_J$trfn = trfn
BioGeoBEARS_run_object_J$geogfn = geogfn
BioGeoBEARS_run_object_J$max_range_size = 4
BioGeoBEARS_run_object_J$include_null_range = TRUE
BioGeoBEARS_run_object_J$num_cores_to_use = 1

# Activamos el parámetro 'j'
BioGeoBEARS_run_object_J$BioGeoBEARS_model_object@params_table["j","type"] = "free"
BioGeoBEARS_run_object_J$BioGeoBEARS_model_object@params_table["j","init"] = 0.0001
BioGeoBEARS_run_object_J$BioGeoBEARS_model_object@params_table["j","est"]  = 0.0001
BioGeoBEARS_run_object_J$BioGeoBEARS_model_object@params_table["j","min"]  = 0.00001
BioGeoBEARS_run_object_J$BioGeoBEARS_model_object@params_table["j","max"]  = 2.99999

print("Optimizando modelo DEC+J...")
resDECJ = bears_optim_run(BioGeoBEARS_run_object_J)

# Extraer parámetros estadísticos
stats_DECJ <- extract_params_from_BioGeoBEARS_results_object(results_object = resDECJ, returnwhat = "table", addl_params = c("j"))

# ==============================================================================
# 4. TABLA COMPARATIVA
# ==============================================================================
print("================ COMPARACIÓN DE MODELOS ================")
tabla_comparativa <- rbind(DEC = stats_DEC, DEC_J = stats_DECJ)
print(tabla_comparativa)
print("========================================================")

¿Cómo leer los resultados de la consola?

El bloque anterior generará una tabla al final de la consola de R. Estos son los significados de las columnas más importantes:

  • d (Dispersión): Tasa que asume una expansión lenta del territorio de la especie.
  • e (Extinción): Tasa de desaparición en áreas locales.
  • j (Jump / Salto): Probabilidad de que se forme una nueva especie mediante un salto de larga distancia a otra isla.
  • LnL (Verosimilitud): Qué tan bien ajusta el modelo (números más cercanos a 0 son mejores).
  • AICc (Criterio de Akaike): El indicador decisivo. El modelo con el número AICc más bajo es la explicación evolutiva correcta.
